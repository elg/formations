L'interpreteur

Shell ?

Interface avec le noyau (Linux dans notre cas)

sh, bash, ksh, zsh...

Bash est l'interpreteur retenu pour Linux

Le Shell et le Terminal sont deux choses différentes

Rappel sur quelques commandes

Se déplacer et observer une arborescence

cd : changer de répertoire

ls : lister le contenu d'un répertoire

find: rechercher dans un répertoire

locate/updatedb : rechercher sur l'ensemble du système

du : observer la taille d'un fichier / d'un répertoire (-sh)

df : observer l'espace disque utilisé sur un système de fichier

Manipulations de fichiers

mkdir : créer un répertoire vide

touch : modifier la date du dernier accès à un fichier
souvent utilisé pour créer un fichier vide

cp : copier un fichier ou un répertoire (-r)

mv : déplacer un fichier ou un répertoire

rm : supprimer un fichier ou un répertoire (-r)

rmdir : supprimer un répertoire vide

ln : créer un lien (éventuellement symbolique -s)

chmod : changer les droits sur un fichier

chown : changer le propriétaire d'un fichier

Les éditeurs / lecteurs

vim / vi : éditeur avancé en mode texte

nano : éditeur simple en mode texte

more : affichage d'un fichier texte page par page

less : affichage avancé d'un fichier texte

Gestion des processus et des informations machines

ps : liste des processus en cours d'execustion(ps -aux)

top : affichage avancé de la liste des processus et infos

htop : affichage très avancé de la liste des processus et infos

uptime : affichage de la durée écoulé depuis le démarrage et de la charge

date : affiche la date (+%Y%m%d par exemple)

time : affiche la durée de l'éxécution d'une commande

Les entrées et sorties

Définitions

entrée standard : entrée utilisée par les commandes pour obtenir des données
Défaut : clavier.

sortie standard : sortie utilisée par les commandes pour afficher un résultat
Défaut : le terminal(l'écran).

sortie d'erreur : sortie utilisée par les commandes pour afficher les erreurs
Défaut : le terminal (l'écran)

Les redirections

> : redirection de la sortie standard vers un fichier (écrase le fichier)

>> : redirection de la sortie standard vers la fin d'un fichier (ajoute)

2> : redirection de la sortie d'erreur vers un fichier

&> : redirection de la sortie d'erreur et de la sortie standard vers un fichier

< : redirection de l'entrée standard

| : redirection de la sortie standard d'une commande vers l'entrée standard d'une autre commande

Script Shell

Définition

Suite de commandes dans un fichier texte

Peux contenir des structures conditionelles

Peux contenir des variables

Peux contenir des boucles

Chaque ligne débutant par un "#" est un commentaire (non exécuté)

La première ligne est une exception, elle commence par #:
#!/bin/bash

TP 1 : écrire un script qui créer un répertoire "monrep" et un fichier "monrep/monfic"

Quelques règles utiles

Indenter son code (cf algorithmique)

Commenter les blocs de code (ne jamais hésiter à commenter)

Utiliser des noms de fonctions et variables compréhensibles (pas de a, b, c)

Les paramètres d'un script

./monscript.sh param1 param2

$0 == "./monscript.sh"

$1 == "param1"

$2 == "param2"

$@ == "param1 param2"

Syntaxe

variable et boucles

Les variables ne sont pas typées (chaînes de caractères)

Test conditionel : if [ montest ]; then ...; fi

Boucle pour : for i in {1 .. 10}; do echo $i; done

Boucle tant que : while read line; do echo $line; done < monfic.txt
